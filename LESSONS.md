# Lessons Log

## Purpose
Capture meaningful corrections, repeated mistakes, failed assumptions, and audit findings that should change future agent behavior in this repository.

## When To Add An Entry
- A user correction reveals a repeated pattern of error.
- A review-agent or audit finding exposes a preventable gap.
- A validation claim was made without enough evidence.
- A prompt, ontology, or schema mistake required a new guardrail.

## Entry Rules
- Keep entries operational and evidence-based.
- Record the smallest durable lesson that would have prevented the issue.
- Link policy-grade lessons back into `AGENTS.md`, `WORKFLOW.md`, or prompt assets when needed.
- Do not log one-off noise, personal reflection, or vague advice.
